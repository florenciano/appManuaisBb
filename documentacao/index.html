Objetivo do Projeto
Transpor os manuais do Blackboard da versão .pdf para .html mantendo a mesma (ou a mais próxima de) experiência de uso do modelo de origem. E a produção de novas demandas conforme necessidade.

Requisitos mínimos do Projeto
Responsive web design (RWD)*; Cross-browser compatível com todos os browser que recebem suporte do Blackboard (veja lista).
*: Desktop e tablet.

Metas do Projeto

  • Permitir que o usuário tira as principais dúvidas referente ao Blackboard, Catme, Clickers e os Recursos de aprendizagem (casos, simuladores e Cursos online);
  • Facilitar a busca por determinados assuntos de interesses;

Requisitos

As tecnologias utilizadas no desenvolvimento deste projetos foram:

  • HTML5/CSS3
  • Framework de autoração de css: COMPASS. (mais informação)
  • Extensão de linguagem css: SASS. (mais informação)
  • Javascript e biblitoteca de script jQuery.
  • Framework MVW Javascript: AngularJS. (mais informação)

Estrutura de diretórios

A estrutura de diretórios seguiu as boas práticas de desenvolvimento web baseado nas recomendações do boilerplate templantes. O projeto foi configurado no diretório raiz front.

  • Diretório raiz

    Responsável pela locação do index.html.

    Para atender as recomendações do conceito Friendly URLs foram criados novos diretórios para cada tema de manual a ser produzido. Para facilitar a identificação destes itens a letra inicial fica gravado em caixa alta.

    Uma vez definido o temas, o próximo passo é novamente agrupar os HTMLs por assunto. Abaixo segue um exemplo com o tema Blackboard.

    E para cada assunto segue a mesma regra para os tópicos existentes.

  • Diretório de imagens

    Ficarão alocados neste diretório todas as imagens necessárias para o desenvolvimento do projeto.

    O armazenamento das imagens seguem o mesmo critério adotado na padronização de estruturação dos documentos HTML´s. O agrupamento segue por temas, assuntos e tópicos.

    A convenção adotado para organizar as imagens do projeto foi rotular os arquivos PSDs com _stepX.psd (conforme indicado no box amarelo), indicando assim a sequencia dos passos das explicações dos manuais. Na exportação das imagens manteve-se o mesmo padrão da nomenclatura, porém, sem o sinal de '_' (conforme indicado no box azul).

    • Diretório de imagens 'ui'
      Local exclusivo de armazenamento das imagens extraídas da GUI (Graphical User Interface). Os arquivos PSDs que compõe a GUI estão armazenados no diretório do Projeto denominado layouts.

  • Diretório de estilos

    O diretório sass é reponsável por alocar os arquivos .scss que gerarão os documentos .css do projeto. A pasta esta separada em duas divisões:

    • layouts: reponsável pela organização dos principais arquivos .scss do Projeto. Eles geram os arquivos .css necessários para construir a interface (UI) dos manuais. O resultado final da compilação é adicionar no diretório css os seguintes documentos:

    • plugins: responsável pelos arquivos de estilos adicionais para o desenvolvimento do Projeto. Foram utilizados estes dois plugins: _normalizr.scss (ver plugin) e _gridiculous.scss (ver plugin).

  • Diretório de scripts

    Neste diretório ficará os arquivos responsáveis por:
