Objetivo do Projeto
Transpor os manuais do Blackboard da versão .pdf para .html mantendo a mesma (ou a mais próxima de) experiência de uso do modelo de origem.

Requisitos mínimos do Projeto
Responsive web design (RWD); Cross-browser compatível com todos os browser que recebem suporte do Blackboard (veja lista).

Metas do Projeto

  • Permitir que o usuário tira as principais dúvidas referente ao Blackboard por meio da navegação das páginas do projeto;
  • Facilitar a busca por determinados assuntos de interesses;
  • Poder imprimir o documento mantendo a mesma experiência do formato original .pdf;

Estrutura do Projeto

A estrutura de diretórios foi definida com base nas recomendações de boas práticas para o desenvolvimento de projetos web. A estrutura inicial seguiu as diretrizes do boilerplate templantes disponível para desenvolvimento front-end, no entanto, para atender as necessidades peculiares do Projeto ocorreu pequenas modificações. Segue abaixo a estrutura principal de diretórios:

  • Diretório raiz (front)

    Diretório responsável para armazenar o arquivo índice do diretório (index.html). Ainda sobre este diretório fica reservado a alocação de outros arquivos que complementam o desenvolvimento necessário do projeto, por exemplo: favicon.ico.

    Para atender as recomendações do conceito Friendly URLs foram criados novos diretórios para cada tema de manual a ser produzido. Para facilitar a identificação destes itens a letra inicial fica gravado em caixa alta.

    Uma vez definido o temas, o próximo passo é novamente agrupar os HTMLs por assunto. Abaixo segue um exemplo com o tema Blackboard.

    E para cada assunto segue a mesma regra para os tópicos existentes.

    Os documentos HTML´s tem o mesmo none: index.html

  • Diretório de imagens (img)

    Ficarão alocados neste diretórios todas as imagens necessárias para o desenvolvimento do projeto. Os dois principais temas são:

    • Diretório de imagens 'ui'
      Local exclusivo de armazenamento das imagens da GUI (Graphical User Interface). O local de armazenamento da GUI estão armazenados em diretório fora da raiz do Projeto denominada 'layouts'.

    • Diretório de imagens 'geral'
      Local de armazenamento default das imagens dos Manuais. Estão alocados dentro de seus respectivos: temas/assuntos/tópcios. A regra adotada é numerar sequencialmente as imagens de acordo com a ordem de leitura do conteúdo. ex: step1.png; step2.png... stepn.png.

      Para facilitar futuramente quaisquer necessidades de alterações das imagens uma cópia dos arquivos em formato aberto de edição '.psd' foi adicionado no mesmo diretório seguindo a mesma regra de nomenclatura descrita acima. Por convenção acrescentamos um sinal de underline '_' no início do nome. Ex: _step1.psd; _stpen.psd. No entanto, recomendamos não subir para o repositório tais documentos.

    O armazenamento das imagens seguem o mesmo critério adotado na padronização de estruturação dos documentos HTML´s. Na prática, há um agrupamento por temas, por assuntos, e por tópicos.

  • Diretório de estilos (sass)

    Neste diretório ficarão alocados os arquivos .scss que gerarão os documentos .css do projeto. A pasta esta separada em duas divisões:

    • layouts: reponsável pela organização dos arquivos .scss principais do Projeto. Eles geram os .css necessários para construir a interface dos documentos.

    • plugins = responsável pelos arquivos de estilos adicionais para o desenvolvimento do Projeto. No momento estão sendo utilizados: _normalizr.scss (ver plugin) e _gridiculous.scss (ver plugin).

    Compilando .scss para .css

    Para gerar os arquivos .css segue as orientações a seguir:

    • Abra o programa Command Prompt with Ruby;
    • Acesse o diretório d: em seguida <Enter>;
    • Digite o comando cd <diretório do Projeto>;
    • Por fim o comando: compass watch;
