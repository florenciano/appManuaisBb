Objetivo do Projeto
Transpor os manuais do Blackboard da versão .pdf para .html mantendo a mesma (ou a mais próxima de) experiência de uso do modelo de origem.

Requisitos mínimos do Projeto
Responsive web design (RWD); Cross-browser compatível com todos os browser que recebem suporte do Blackboard (veja lista).

Metas do Projeto

  • Permitir que o usuário tira as principais dúvidas referente ao Blackboard por meio da navegação das páginas do projeto;
  • Facilitar a busca por determinados assuntos de interesses;
  • Poder imprimir o documento mantendo a mesma experiência do formato original .pdf;

Estrutura do Projeto

A estrutura de diretórios foi definida com base nas recomendações de boas práticas para o desenvolvimento de projetos web. A estrutura inicial seguiu as diretrizes do boilerplate templantes disponível para desenvolvimento front-end, no entanto, para atender as necessidades peculiares do Projeto ocorreu pequenas modificações. Segue abaixo a estrutura principal de diretórios:

  • Diretório raiz (front)

    Diretório responsável para armazenar o arquivo índice do diretório (index.html). Ainda sobre este diretório fica reservado a alocação de outros arquivos que complementam o desenvolvimento necessário do projeto, por exemplo: favicon.ico.

    Para atender as recomendações do conceito Friendly URLs foram criados novos diretórios para cada tema de manual a ser produzido. Para facilitar a identificação destes itens a letra inicial fica gravado em caixa alta.
